# ADR-0010: Использовать Rust для реализации MVP

Статус: accepted
Дата: 2026-03-13

## Контекст

Перед началом реализации нужно было выбрать основной язык и toolchain для MVP.

Рассматривались варианты:

- Go
- Rust

Требования:

- язык должен подходить для CLI-утилиты
- он должен быть доступен в текущем окружении разработки
- он должен быть удобен для модульной архитектуры и тестов
- он должен нормально работать с shell-интеграциями, файловой системой и
  структурированными runtime-артефактами

## Решение

Для MVP используется Rust.

Причины:

- `rustc` и `cargo` уже доступны в текущем окружении
- Rust хорошо подходит для CLI tooling
- язык дает сильную типизацию для конфигов, runtime-артефактов и переходов
  состояний
- test tooling и package layout подходят для планируемой структуры проекта

Go в MVP не используется, так как в текущем окружении он не установлен.

## Последствия

Плюсы:

- можно сразу начать реализацию без подготовки toolchain
- удобно моделировать контракты данных и ошибки
- хорошо подходит для постепенного наращивания качества и тестов

Минусы:

- shell orchestration и прототипирование местами будут многословнее, чем в Go
- для части Linux-first аудитории Go мог бы быть привычнее
