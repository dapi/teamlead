# doc-review-parallel

Используй этот skill, когда нужно быстро прогнать несколько независимых
документных ревью параллельно, а потом собрать единый итоговый отчет.

Если пользователь прямо просит проверить документацию, не расписывай план,
а сразу запускай `./scripts/review-docs.sh` с подходящими путями.

## Быстрый запуск

Из корня репозитория:

```bash
./scripts/review-docs.sh
```

По умолчанию launcher проверяет только `README.md` и `docs`. Каталог
`.ai-teamlead` жестко исключен из review scope и не должен передаваться в
таргеты.

Для конкретных документов:

```bash
./scripts/review-docs.sh README.md docs/features/0003-agent-launch-orchestration
```

## Что делает launcher

Скрипт `./scripts/review-docs.sh`:

1. запускает четыре параллельных `codex exec`-агента
2. каждому агенту задает отдельный фокус:
   `gaps`, `completeness`, `contradictions`, `consistency`
3. сохраняет артефакты в `.docvalidate/reviews/<timestamp>/`
4. после завершения запускает синтезирующий проход и пишет `summary.md`

## Как использовать в агентской сессии

1. Запусти `./scripts/review-docs.sh` с нужными путями.
2. Дождись завершения.
3. Открой `summary.md` и при необходимости отдельные aspect-отчеты.
4. Верни пользователю короткую сводку с главными findings и ссылкой на каталог
   артефактов.

## Полезные опции

- `--aspects gaps,contradictions`
- `--out-dir .docvalidate/reviews/manual-run`
- `--model gpt-5`
- `--reasoning low`
- `--timeout-seconds 180`
- `--no-synthesis`

## Правила

- Это reviewer-only workflow: не меняй документацию без отдельного запроса.
- Если пользователь просит проверить только часть docs, всегда передавай явные
  пути вместо дефолтного полного набора.
- Никогда не передавай `.ai-teamlead` в review scope: launcher его игнорирует
  намеренно.
- Если запуск упал на одном из аспектов, сначала проверь соответствующий
  `*.log`, потом уже перезапускай весь review.
