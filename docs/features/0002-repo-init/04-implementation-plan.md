# Feature 0002: План реализации

Статус: draft
Последнее обновление: 2026-03-13

## Назначение

Этот документ задает порядок реализации `ai-teamlead init`.

## Порядок работ

1. Зафиксировать feature-спеку и ADR.
2. Переименовать черновой `bootstrap` в `init` на уровне CLI и кода.
3. Реализовать вычисление project-local путей в `./.ai-teamlead/`.
4. Добавить встроенные шаблоны:
   - `settings.yml`
   - `README.md`
   - `flows/issue-analysis-flow.md`
5. Реализовать идемпотентную запись файлов без перезаписи.
6. Добавить unit tests на path resolution и write-if-missing поведение.
7. Добавить integration test на `ai-teamlead init` в тестовом git-репозитории с
   настроенным GitHub `origin`.
8. Синхронизировать `README` и связанные feature/ADR с итоговым контрактом.

## Критерий завершения

Feature можно считать реализованной, если:

- команда называется `ai-teamlead init`
- она создает минимальный versioned project contract layer
- она не трогает `.git/.ai-teamlead/`
- повторный запуск идемпотентен
- тесты проходят
