# issue-analysis staged prompts

Этот каталог содержит staged prompts для анализа issue по трем осям.

Порядок выполнения:

1. `01-what-we-build.md`
2. `02-how-we-build.md`
3. `03-how-we-verify.md`

Правила:

- сначала собрать продуктовую ось
- затем техническую ось
- затем verification-ось
- на каждой оси сначала заполнить `core`, потом добрать релевантные
  `conditional`, затем при необходимости `scaling`
- при выборе секций учитывать тип задачи, размер задачи и тип проекта
- не создавать лишние документы сверх минимального SDD-комплекта без реальной
  необходимости
