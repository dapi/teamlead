# ADR-0001: Repo-local `./.ai-teamlead/settings.yml`

Статус: accepted
Дата: 2026-03-13

## Контекст

`ai-teamlead` проектируется как инструмент, который должен легко подключаться к
любому GitHub-репозиторию. При этом текущее репо используется как dogfooding-
среда, где инструмент разрабатывается и применяется на самом себе.

Нужно было принять решение:

- где должен находиться основной конфиг
- должен ли он быть глобальным или repo-local
- как поддержать независимую работу с несколькими репозиториями одновременно

## Решение

Основной конфиг проекта называется `settings.yml` и хранится в каталоге
`./.ai-teamlead/` целевого репозитория.

Каждый репозиторий имеет свой собственный repo-local конфиг и может запускать
свой независимый экземпляр `ai-teamlead`.

Repo context и GitHub target для MVP берутся из текущего репозитория, в котором
запущен инструмент.

`./.ai-teamlead/settings.yml` не должен переопределять `owner/repo` в первой
версии, чтобы не допускать split-brain между кодовой базой и GitHub-целью.

## Последствия

Плюсы:

- инструмент легко применять к разным репозиториям
- конфиг версионируется вместе с репозиторием
- разные репозитории могут иметь разные project id, статусы и runtime-настройки
- можно одновременно запускать независимые экземпляры на нескольких
  репозиториях

Минусы:

- нужно следить за консистентностью конфигов между репозиториями
- изменения общего контракта конфига нужно аккуратно распространять
- нельзя использовать один репозиторий как runtime-контекст для работы с другим
  GitHub-репозиторием
