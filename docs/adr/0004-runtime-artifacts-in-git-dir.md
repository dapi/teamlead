# ADR-0004: Repo-local runtime-артефакты в `.git/.ai-teamlead/`

Статус: accepted
Дата: 2026-03-13

## Контекст

Для первой версии `ai-teamlead` нужно было определить, где хранить
repo-local runtime-артефакты.

Требования:

- runtime-артефакты должны быть привязаны к конкретному репозиторию
- разные репозитории не должны пересекаться между собой
- временные файлы не должны засорять рабочее дерево
- runtime-артефакты не должны путаться с versioned config

Рассматривались варианты:

- отдельная директория в рабочем дереве, например `.ai-teamlead/`
- директория внутри `.git`
- глобальная директория пользователя

## Решение

Repo-local runtime-артефакты хранятся в `.git/.ai-teamlead/`.

В этой директории допускается хранить:

- lock-файлы
- временные prompt-файлы
- временные артефакты анализа
- диагностические следы текущего запуска

Основной конфиг `./.ai-teamlead/settings.yml` при этом остается versioned
частью репозитория.

## Последствия

Плюсы:

- runtime-артефакты остаются repo-local
- рабочее дерево не засоряется временными файлами
- разные репозитории автоматически изолированы друг от друга
- config и runtime state четко разделены

Минусы:

- часть артефактов оказывается менее заметной для ручного просмотра
- при некоторых сценариях диагностики нужно отдельно заглядывать в `.git`
