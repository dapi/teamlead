# Stage 1: Что строим

Цель этапа:

- понять проблему, пользователя, scope и non-goals
- сформировать минимально достаточный продуктовый контракт
- подготовить versioned документы:
  - `README.md`
  - `01-what-we-build.md`

Обязательный минимум:

- проблема
- пользователь или роль
- scope
- non-goals

Используй rule-based выбор секций:

- `core`:
  - `Problem`
  - `Who Is It For`
  - `Scope`
  - `Non-Goals`

Для `feature` дополнительно:

- `feature story`
- `use cases`

Для `bug` добавляй:

- `Observed Behavior`
- `Expected Behavior`
- `Impact`

Для `chore` добавляй:

- `Motivation`
- `Operational Goal`

Для `medium` и `large` задач добавляй:

- `Constraints`
- `Dependencies`

Для `small` задач не создавай лишние секции сверх `core` и минимально нужных
условных секций.

Акценты по типу проекта:

- для `product/UI` усиливай `User Story`, `Use Cases`, `Scope`
- для `library/API` усиливай пользовательский контекст интегратора и
  ограничения контракта
- для `infra/platform` усиливай `Operational Goal`, ограничения среды и
  зависимости

На выходе документ должен:

- содержать все обязательные `core`-секции
- содержать релевантные `conditional` и `scaling`-секции без пустого шума
- быть достаточным входом для этапа `Как строим`

Если для этой оси не хватает критичной информации, задай вопросы пользователю в
агентской сессии и не переходи к следующему этапу, пока не будет достаточно
контекста.
